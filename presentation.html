Current Situation

HR recruitment relied on spreadsheets, paper forms, and manual email tracking. This caused data duplication, delays, and reporting difficulties.

Key Problems

  • No centralized applicant database
  • Slow screening & tracking
  • Limited analytics & reporting
  • High risk of data inconsistency

Need for Digital HR

Digital HR systems improve efficiency, accuracy, accountability, and decision-making through automation and analytics.

üéØ Objectives of the Study

General Objective

Design and develop a secure, database-driven, web-based Recruitment and Applicant Management System with role-based access.

Specific Objectives

  • Applicant Web Form (Tier-1)
  • Secure Login & RBAC
  • HR Dashboard with CRUD
  • Bulk CSV Upload
  • Analytics & Reports
  • System Logs & Security

üìä ISO/IEC 25010 Software Quality Evaluation

Functional Suitability

Evaluates whether the system provides all required recruitment functionalities accurately and appropriately.

  • Applicant Web Form & Pre-screening
  • CRUD operations for applicants
  • Bulk CSV upload and validation
  • Analytics dashboard and logs

Performance Efficiency

Measures system responsiveness, processing speed, and handling of large datasets.

  • Fast dashboard rendering
  • Optimized MySQL queries
  • Efficient bulk upload processing

Usability (Interaction Capability)

Assesses ease of use, learnability, and user satisfaction.

  • Clear navigation structure
  • Form validation and error prompts
  • HR-friendly dashboard layout

Reliability

Determines system stability, fault tolerance, and data consistency.

  • Input validation mechanisms
  • Error handling during uploads
  • System logs for traceability

Security

Evaluates protection of sensitive data and access control.

  • Role-Based Access Control (RBAC)
  • Password hashing and secure sessions
  • SQL injection prevention
  • Intranet-only deployment

Compatibility

Ensures the system works reliably across supported environments.

  • Chrome, Edge, Firefox compatibility
  • Co-existence within company intranet

Maintainability

Assesses ease of modification, debugging, and enhancement.

  • Modular backend APIs
  • Reusable frontend components

Portability & Scalability

Evaluates adaptability to increased users and data volume.

  • Scalable database structure
  • Expandable recruitment modules
